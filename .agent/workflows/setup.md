---
description: プロジェクトの初期設定を行い、不要なファイルを削除します
---
このコマンドは、テンプレートから新しく作成されたプロジェクトの初期セットアップを行うためのコマンドです。

AIエージェントは、以下の手順で現在のプロジェクトの初期化を行ってください。

1. **ロードマップの初期化**
   - `docs/status/roadmap.md` の内容が、テンプレートの初期値（例: `# [Project Name] Roadmap` というプレースホルダーが含まれている等）の場合、またはファイルが存在しない場合は、`.agent/templates/roadmap.md` をベースにして現在のプロジェクト用に新しく作成・上書きしてください。

2. **技術スタックの特定と規約の準備**
   - ディレクトリ内の主要ファイル（`package.json`, `build.gradle`, `pyproject.toml` 等）を走査し、プロジェクトの技術スタック（Android, Next.js, Python 等）を特定してください。
   - **階層化構成の導入**:
     - `.agent/custom/` ディレクトリを作成してください。
     - 特定した技術スタックに応じた固有の規約（Compose 活用方針、Lint ルール等）がある場合は、`.agent/custom/` 配下に新しい規約ファイル（例: `android_rules.md`）を作成し、プロジェクト固有のコンテキストとして維持することをユーザーに提案してください。

3. **テンプレート固有ファイルのクリーンアップ**
   - テンプレートの説明が入っているルートの `README.md` を、現在のプロジェクト名と簡単な説明のみを記述した状態に書き換えてください。
   - テンプレートリポジトリに対する貢献ガイドである `CONTRIBUTING.md` を削除してください。
   - テンプレートリポジトリ固有のライセンスである `LICENSE` ファイルを削除してください（プロジェクト独自のライセンスが必要な場合は、ユーザーに確認してください）。

4. **環境構築スクリプトの実行リマインド**
   - 以下のスクリプトを実行して、GitHub ラベルと Git フックを有効化することをユーザーに推奨、または許可を得て実行してください。
     - `./scripts/windows/setup-labels.ps1`
     - `./scripts/windows/setup-hooks.ps1`

5. **セットアップ完了の報告**
   - 以上の手順が完了したら、ユーザーに初期化が完了したことと、作成・削除・変更したファイルの一覧を報告してください。
   - また、「次はロードマップ（`docs/status/roadmap.md`）に最初のタスクを記述して開始しましょう」と案内してください。
