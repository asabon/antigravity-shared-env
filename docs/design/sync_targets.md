# 同期対象ファイル一覧 (Sync Targets)

本ドキュメントでは、初期セットアップ（`setup-workflow`）および継続同期（`sync-workflow`）において、どのファイルがどのように管理・同期されるかの全体像を定義します。

---

## 🚦 基本原則
同期は **「共通基盤の強制適用（Core）」** と **「プロジェクト設定の確認・選択（Settings）」**、そして **「ユーザー資産の保護（Exclude）」** に分類されます。

---

## 📂 カテゴリ別一覧

| 対象パス | setup (初期) | sync (継続) | 主な役割・目的 / 備考 |
| :--- | :---: | :---: | :--- |
| **`.agent/custom/`** | 〇 (READMEのみ) | ❌ | **プロジェクト独自の規約**（同期対象外）。 ※2 |
| **`.agent/rules/`** | 〇 | 〇 | 全プロジェクト共通の絶対規約（`01_workflow.md` 等） |
| **`.agent/skills/`** | 〇 | 〇 | コマンド（`flow-kickoff`, `git` 等）の安全マニュアル |
| **`.agent/templates/`** | 〇 | 〇 | 新規作成時の `task.md` などの素・ひな形 |
| **`.agent/workflows/`** | 〇 | 〇 | `/dev-start` や `/cleanup` を呼ぶ自動実行スクリプト |
| **`.antigravityrule`** | 〇 | 〇 | AIエージェントの動作指針、ステータスバー表示規則 |
| **`.github/`** | 〇 | 〇 | PR/Issueテンプレート、ReleaseDrafter、共通CI |
| **`.gitignore`** | 〇 | 💬 | 共通除外設定。 ※1 |
| **`.hooks/`** | 〇 | 〇 | Gitフックのテンプレート（`pre-commit`, `pre-push` 等） |
| **`.vscode/settings.json`** | 〇 | 💬 | コマンドの自動実行許可等。 ※1 |
| **`docs/` (上記以外)** | ❌ | ❌ | ロードマップ等の進捗、アプリ固有の仕様など |
| **`docs/design/`** | 〇 | 〇 | 本設計書一式（操作マニュアル） |
| **`docs/workflow/`** | 〇 | 〇 | 人間用操作マニュアル |
| **`scripts/`** | 〇 | 〇 | 初期導入時の GitHooks 有効化スクリプト |

---

※1: `sync-workflow` 実行時は、差分を確認して選択的に反映（パッチ等）を推奨します。
※2: 初期セットアップ時のみテンプレート枠（README）が配信され、以降は上書き等から同期保護されます。
