# 共通環境同期ワークフロー (sync-env)

このワークフローは、`env/` 内のテンプレートから必要なアセットを取り込む（Pull）だけでなく、プロジェクト側で行われたルールの改善を共通リポジトリへ還元する（Feedback）自律的なプロセスを提供します。

## 手順

### 1. プロジェクト分析
- `env/README.md` を読み、提供されているアセットの全体像を把握してください。
- 現在のリポジトリの構造、主要言語を分析し、対象スタック（Android, TypeScript, Python）を特定してください。

### 2. ルールの改善検知と還元（Feedback） [PRE-SYNC]
同期を実行する前に、現在のプロジェクトで既に行われた改善を保護・還元します。

1. **差分の比較**:
   - `env/` 内のテンプレートと、現在プロジェクトのルート（`.agent/rules/` 等）にあるファイルを比較してください。
2. **改善箇所の特定**:
   - 単なるプロジェクト名の変更やパスの修正ではなく、**「より分かりやすいプロンプトの記述」「不足していた汎用的な制約の追加」**などの汎用的な改善が行われているか特定してください。
3. **還元（Feedback）の提案**:
   - 汎用的な改善が見つかった場合、ユーザーに報告し、それを `env/` 内のテンプレートへ反映するか確認してください。
   - **重要**: 還元する際は、プロジェクト固有の固有名詞をテンプレート変数や汎用的な表現に差し戻す必要があります。

### 3. テンプレートの展開とリネーム（Pull）
特定したスタックに基づき、アセットをプロジェクトルートに展開してください。

- **アンカー設定**: `env/antigravityrule.template` -> `./.antigravityrule`
- **AI ルール**: `env/agent_template/` -> `./.agent/`
- **GitHub**: `env/github_template/workflows/` -> `./.github/workflows/` (スタック別テンプレート含む)
- **Hooks/Scripts**: `env/hooks_template/` -> `./.hooks/`, `env/scripts_template/` -> `./scripts/`

**※ 展開時、必ずドット付きの正しい名前にリネームしてください。**

### 4. 展開後の最適化（Subtractions）
コピー完了後、直ちに「引き算」による最適化を実行してください。

1. **`.antigravityrule` のクリーンアップ**: 不要な技術スタックのインデックスと、`rules:` リスト内の不要なファイルパスを削除してください。
2. **ルールの整理**: `./.agent/rules/` 内の不要なスタック別ルールファイルを削除してください。

### 5. 完了報告
- 還元（Feedback）した改善点と、新たに展開・更新したファイルの一覧をユーザーに報告してください。
- コンフリクトの恐れがある場合や、手動での調整が必要な箇所があれば併せて伝えてください。
